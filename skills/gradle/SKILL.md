# Gradle Skill

This skill provides comprehensive Gradle build automation capabilities for Claude Code, enabling you to work effectively with Gradle projects.

## Capabilities

This skill enables Claude to:
- Initialize new Gradle projects with various project types (Java, Kotlin, Android, etc.)
- Build and compile Gradle projects
- Run tests and generate reports
- Manage dependencies and view dependency trees
- Execute custom Gradle tasks
- Configure and update Gradle wrapper versions
- Debug build issues and optimize build performance

## When to Use This Skill

Use this skill when you need to:
- Create a new project using Gradle as the build tool
- Build, test, or package an existing Gradle project
- Add, update, or troubleshoot dependencies
- Configure build scripts or Gradle settings
- Optimize build performance
- Migrate from other build tools to Gradle
- Debug Gradle build failures

## Project Context

This skill works best with projects that:
- Use Gradle as their build system
- Are Java, Kotlin, Groovy, Scala, or Android projects
- Have a `build.gradle`, `build.gradle.kts`, or `settings.gradle` file
- Follow standard Gradle project structure

## Available Workflows

- **init** - Initialize a new Gradle project
- **build** - Build and compile the project
- **test** - Run tests and generate reports
- **dependencies** - Manage and inspect dependencies
- **tasks** - List and execute Gradle tasks
- **wrapper** - Manage Gradle wrapper version
